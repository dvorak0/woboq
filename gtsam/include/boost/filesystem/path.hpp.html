176 // ----- character encoding conversions -----
177
178 // Following the principle of least astonishment, path input arguments
179 // passed to or obtained from the operating system via objects of
180 // class path behave as if they were directly passed to or
181 // obtained from the O/S API, unless conversion is explicitly requested.
182 //
183 // POSIX specfies that path strings are passed unchanged to and from the
184 // API. Note that this is different from the POSIX command line utilities,
185 // which convert according to a locale.
186 //
187 // Thus for POSIX, char strings do not undergo conversion. wchar_t strings
188 // are converted to/from char using the path locale or, if a conversion
189 // argument is given, using a conversion object modeled on
190 // std::wstring_convert.
191 //
192 // The path locale, which is global to the thread, can be changed by the
193 // imbue() function. It is initialized to an implementation defined locale.
194 //
195 // For Windows, wchar_t strings do not undergo conversion. char strings
196 // are converted using the "ANSI" or "OEM" code pages, as determined by
197 // the AreFileApisANSI() function, or, if a conversion argument is given,
198 // using a conversion object modeled on std::wstring_convert.
199 //
200 // See m_pathname comments for further important rationale.
201
202 // TODO: rules needed for operating systems that use / or .
203 // differently, or format directory paths differently from file paths.
204 //
205 // **********************************************************************************
206 //
207 // More work needed: How to handle an operating system that may have
208 // slash characters or dot characters in valid filenames, either because
209 // it doesn't follow the POSIX standard, or because it allows MBCS
210 // filename encodings that may contain slash or dot characters. For
211 // example, ISO/IEC 2022 (JIS) encoding which allows switching to
212 // JIS x0208-1983 encoding. A valid filename in this set of encodings is
213 // 0x1B 0x24 0x42 [switch to X0208-1983] 0x24 0x2F [U+304F Kiragana letter KU]
214 // ^^^^
215 // Note that 0x2F is the ASCII slash character
216 //
217 // **********************************************************************************
218
219 // Supported source arguments: half-open iterator range, container, c-array,
220 // and single pointer to null terminated string.
221
222 // All source arguments except pointers to null terminated byte strings support
223 // multi-byte character strings which may have embedded nulls. Embedded null
224 // support is required for some Asian languages on Windows.
225
226 // "const codecvt_type& cvt=codecvt()" default arguments are not used because this
227 // limits the impact of locale("") initialization failures on POSIX systems to programs
228 // that actually depend on locale(""). It further ensures that exceptions thrown
229 // as a result of such failues occur after main() has started, so can be caught.
