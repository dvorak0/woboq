29/**
30 * An EliminatableClusterTree, i.e., a set of variable clusters with factors,
31 * arranged in a tree, with the additional property that it represents the
32 * clique tree associated with a Bayes net.
33 *
34 * In GTSAM a junction tree is an intermediate data structure in multifrontal
35 * variable elimination. Each node is a cluster of factors, along with a
36 * clique of variables that are eliminated all at once. In detail, every node k
37 * represents a clique (maximal fully connected subset) of an associated chordal
38 * graph, such as a chordal Bayes net resulting from elimination.
39 *
40 * The difference with the BayesTree is that a JunctionTree stores factors,
41 * whereas a BayesTree stores conditionals, that are the product of eliminating
42 * the factors in the corresponding JunctionTree cliques.
43 *
44 * The tree structure and elimination method are exactly analogous to the
45 * EliminationTree, except that in the JunctionTree, at each node multiple
46 * variables are eliminated at a time.
47 *
48 * \ingroup Multifrontal
49 * \ingroup hybrid
50 * \nosubgrouping
51 */
52class GTSAM_EXPORT HybridJunctionTree
53 : public JunctionTree<HybridBayesTree, HybridGaussianFactorGraph> {
54 public:
55 typedef JunctionTree<HybridBayesTree, HybridGaussianFactorGraph>
56 Base; ///< Base class
57 typedef HybridJunctionTree This; ///< This class
58 typedef std::shared_ptr<This> shared_ptr; ///< Shared pointer to this class
59
60 /**
61 * Build the elimination tree of a factor graph using precomputed column
62 * structure.
63 * @param factorGraph The factor graph for which to build the elimination tree
64 * @param structure The set of factors involving each variable. If this is
65 * not precomputed, you can call the Create(const FactorGraph<DERIVEDFACTOR>&)
66 * named constructor instead.
67 * @return The elimination tree
68 */
69 HybridJunctionTree(const HybridEliminationTree& eliminationTree);
70};
